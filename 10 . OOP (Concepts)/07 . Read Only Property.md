A **read-only property** is a property that:

- **Can be read (Get)**
    
- **Cannot be modified (no Set)**
    

 This is useful when you want to **protect data** from being changed outside the class, but still allow other parts of the program to access it.
 
# When to Use Read-Only Properties?

- Constants that shouldn’t change (e.g., **ID, Birthdate**)
    
- Computed values (e.g., **Area = πr²**)
    
- Values set **once** in constructor and then fixed

```cpp
#include <iostream>
using namespace std;


class clsPerson

{

private:

    string _FirstName;

    string _LastName;

    int _ID = 10;

public :

    // Property Get is the read only because there's no Set property .

    int GetID()

    {
        return _ID;
    }

  

    // Property Set :

    void SetFirstName(string FirstName)

    {
        _FirstName = FirstName;
    }

  

    // Property Get :

    string FirstName()

    {
        return _FirstName;
    }

  

    // Property Set :

     void SetLatName(string LastName)

    {
        _LastName = LastName;
    }

  

    // Property Get :

    string LastName()

    {

        return _LastName;

    }

  

    // Property Get :

    string FullName()

    {

        return _FirstName + " " + _LastName;

    }

  

};

  

int main()

{

  

    clsPerson Person1;

  

    Person1.SetFirstName("Ualid");

    Person1.SetLatName("Ahlidou");

  

    cout << "First Name :" << Person1.LastName() << endl;

    cout << "Last Name :" << Person1.LastName() << endl;

    cout << "Full Name :" << Person1.FullName() << endl;

    cout << "ID = " << Person1.GetID() << endl;

  

    system("pause>0");

    return 0;

}
```
