# **Objects** and Dynamic <span style="color:rgb(0, 176, 80)">Array</span> 




### - Example :

```cpp
#include<iostream>
#include<vector>

using namespace std;

class clsA
{
public:

	// dummy constructor
	clsA() {}

	//Parametarized Constructor
	clsA(int value)
	{
		x = value;
	}

	int x;

	void Print()
	{
		cout << "The value of x=" << x <<endl;
	}

};


int main() 
{
	short NumberOfObjects = 5;

	// allocating dynamic array
   // of Size NumberOfObjects using new keyword

	clsA * arrA = new clsA[NumberOfObjects];

	// calling constructor
	// for each index of array
	for (int i = 0; i < NumberOfObjects; i++) {
		arrA[i] = clsA(i);
	}

	// printing contents of array
	for (int i = 0; i < NumberOfObjects; i++) {
		arrA[i].Print();
	}

	return 0;

	
	system("pause>0");

}

```


---

### 1. What is a dynamic array?

- Normally, if you write:
    
  ```cpp
    clsA arrA[5];
    ```
    
    The array size (`5`) is **fixed at compile time** (static array).
    
- If you don’t know the size at compile time (e.g., user input, variable), you need a **dynamic array**, created on the **heap** using `new`:
    
  ```cpp
  clsA* arrA = new clsA[NumberOfObjects];
    ```
    
    Now, `arrA` is a pointer that points to the first element of a block of objects in memory.
    

---

### 2. What happens when you write `new clsA[NumberOfObjects]`?

- C++ will allocate a continuous block of memory big enough to hold `NumberOfObjects` objects of type `clsA`.
    
- It then **calls the default constructor** (`clsA() {}`) for each element.  
    That’s why you **must** have a default constructor if you use this form.
    

So after this line:

```cpp
 clsA* arrA = new clsA[NumberOfObjects];
```

You have 5 objects of type `clsA` in memory, each constructed using the default constructor.

---

### 3. How do you give values to them?

Since all were built with the **default constructor**, they don’t yet have `x` set.  
You then assign new values in a loop:

```cpp
for (int i = 0; i < NumberOfObjects; i++) {
    arrA[i] = clsA(i);  // creates a temporary object with x=i, then assigns it
}
```

Here’s what happens for `i = 0`:

- `clsA(i)` → calls the parameterized constructor and builds a **temporary object** with `x = i`.
    
- `arrA[i] = ...` → uses the assignment operator to copy that temporary into the object already constructed at index `i`.
    

So the default-constructed object gets **overwritten**.

---

### 4. Printing

```cpp
 for (int i = 0; i < NumberOfObjects; i++) {
    arrA[i].Print();
}
```

Now each object prints the value you assigned.

---

### 5. Memory cleanup

Since you used `new[]`, you should also free the memory with `delete[]`:

```cpp
 delete[] arrA;
```

Otherwise, you get a memory leak.

---


✅ **Summary of how your code works:**

1. `new clsA[NumberOfObjects]` allocates 5 objects (calls default constructor).
    
2. You overwrite each with a temporary built using the parameterized constructor.
    
3. You print them.
    
4. You should `delete[] arrA;` at the end.
