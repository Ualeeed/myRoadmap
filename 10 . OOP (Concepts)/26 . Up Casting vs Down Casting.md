Explanation Example :  

```cpp
#include <iostream>

using namespace std;

class clsPerson
{

public:

    string FullName="Mohammed Abu-Hadhoud";
 
};

class clsEmployee : public clsPerson
{

public:
    string Title = "CEO";

};

int main()

{

    clsEmployee Employee1;
  
    cout << Employee1.FullName << endl;

    ////upcasting
    //this will convert employee to person.
    clsPerson * Person1 = &Employee1;
    cout << Person1->FullName << endl;
   

    //clsPerson Person2;
    //cout << Person2.FullName << endl;

    ////downcasting : you cannot convert person to employee
    //clsEmployee* Employee2 = &Person2;



    system("pause>0");
    return 0;
}
```

##### 1. **Upcasting : 

- Converting a **derived class (sub)** object to a **base class (Super)** type.
- You **lose access** to members that only belong to the derived class. 
 - Example :

```cpp
clsEmployee Employee1;      // Derived class object
clsPerson* Person1 = &Employee1; // Upcasting

```

- `Employee1` has both `FullName` (from `clsPerson`) and `Title` (from `clsEmployee`).
    
- But once we up cast to `clsPerson*`, you can only access **`FullName`** (because `clsPerson` doesn’t know about `Title`).
- So : 
```cpp
cout << Person1->FullName; // ✅ Works
cout << Person1->Title;    // ❌ Error (Title not in clsPerson)

```


##### 2. **Down casting : 

- Converting a **base class (Super)** object to a **derived class (Sub)** type.
- down casting is not safe and not allowed.

```cpp
clsPerson Person2;                   // Base class object
clsEmployee* Employee2 = &Person2;   // Downcasting (not allowed!)

```

Here `Person2` **doesn’t have `Title`**, so treating it as `clsEmployee` is invalid.  
If you try to access `Employee2->Title`, the program will crash or behave unpredictably.
