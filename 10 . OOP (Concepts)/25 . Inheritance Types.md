#### 1. <span style="color:rgb(20, 192, 255)">Single</span> Inheritance : 

```mermaid
graph TD
    A[Class A] --> B[Class B]

```

Class A inherit Class B.

#### 2. <span style="color:rgb(20, 192, 255)">Multilevel</span> Inheritance : 

```mermaid
graph TD
    A[Class A] --> B[Class B] --> C[Class C]

```


- Class C inherit Class B
- Class B inherit Class A 
- so automatically Class C inherit Class A 


#### 3. <span style="color:rgb(20, 192, 255)">Hierarchical</span> Inheritance : 

```mermaid
graph TD
    A[Class A] --> B[Class B]
    A[Class A] --> C[Class C]
    A[Class A] --> D[Class D] 

```

- Classes B, C & D inherit Class A


#### Type that exist only in C++ : 


#### 1. <span style="color:rgb(255, 66, 66)">Multiple</span> Inheritance : 

```mermaid
graph TD
    A[Class A] --> C[Class C]
    B[Class B] --> C[Class C]

```

- Class C inherit Class A & B 
- this type of inheritance need a lot of focus because they cause problem 
- like if we have a function named ( **Print()** ) in both Class A and Class B and we Override the function inside the Class C it will causes problems.

#### 2. <span style="color:rgb(255, 66, 66)">Hybrid</span> : 

```mermaid
graph TD
    A[Class A] --> B[Class B]
    A[Class A] --> C[Class C]
    B[Class B] --> D[Class D]
    C[Class C] --> D[Class D]

```

- ### Problem:

Class `D` will inherit **two copies of `A`** (ambiguity problem).
