A **pure virtual function** in C++ is a function that:

- Has **no implementation** in the base class. (implementation is the body)
    
- Is declared by writing `= 0` at the end.
    
- Forces derived classes to **override it**.


- Syntax : 

```cpp
 class Base {
 public:
   virtual void FunctionName() = 0; // pure virtual function
 };
```

- Example : 

```cpp

#include <iostream>
using namespace std;


//Abstract Class / Interface / Contract.
class clsMobile
{
    virtual void Dial(string PhoneNumber) = 0;
    virtual void SendSMS(string PhoneNumber,string Text) = 0;
    virtual void TakePicture() = 0;

};

class clsiPhone : public clsMobile
{

  //This class signed a contract with clsMobile abstract class   therefore it should implement everything in the abstract class
```

###  **Pure Virtual Functions** 

- **Pure Virtual Functions** Also known as **Abstract Classes** or **Interface** 

In Our code, inside `clsMobile`, you see functions like this:

```cpp
 virtual void Dial(string PhoneNumber) = 0;
```

- `= 0` means this function **has no implementation** in the base class.
    
- It is just a **declaration**, forcing derived classes to **provide their own implementation**.
    
- This is called a **pure virtual function**.
- A class that contains **at least one pure virtual function** becomes an **abstract class**.
- Abstract classes **cannot be instantiated directly** (you cannot create objects from them).
    
- They act like a **contract** or **blueprint** for other classes.

```ad-attention
title: Summary 
"Any class that inherits from me MUST provide its own version of this function."
```

**Summary**

- **Pure virtual function** → `= 0`, no implementation, must be overridden.
    
- **Abstract class** → contains at least one pure virtual function, cannot be instantiated.
    
- **Interface in C++** → Abstract class with only pure virtual functions.
