- Access modifiers (or access specifiers) are keywords in object-oriented languages that set the accessibility of classes, methods, and other members. 

- Example of Access modifiers : 

```cpp
#include <iostream>

using namespace std;

class clsA
{
private:
    //only accessible   inside this class, neither derived classes nor outside class.
    int _Var1;
    void _Fun1()
    {
        cout << "Function 1";
    }

protected:
    //only accessible  inside this class and all derived classes, but not outside class
    int Var2;
    void Fun2()
    {
        cout << "Function 1";
    }

public:
    // Accessable inside this class, all derived classes, and outside class
    int Var3;
    void Fun3()
    {
        cout << "Function 1";
    }

};

class clsB : public clsA
{
    
public:

    void Func1()
    {
        cout << clsA::Var2;
    }
};

int main()

{

   
    system("pause>0");
    return 0;
}

```


- <span style="color:rgb(0, 176, 80)">clsA</span> is the **base class** with members under:
    
    - `private` → accessible **only inside <span style="color:rgb(0, 176, 80)">clsA</span>
        
    - `protected` → accessible **inside <span style="color:rgb(0, 176, 80)">clsA</span> and derived classes**, but **not outside**
        
    - `public` → accessible **everywhere** (inside, derived classes, outside)
        
- <span style="color:rgb(0, 176, 80)">clsB</span> inherits publicly from <span style="color:rgb(0, 176, 80)">clsA</span>.

- This means:
- 
        
        - ` clsA::public ` → stays **public** in `clsB`
            
        - ` clsA::protected ` → stays **protected** in `clsB`
            
        - ` clsA::private ` → remains **inaccessible**
            
- Inside `clsB::Func1()`, you access `clsA::Var2`, which is `protected`, so it’s valid.


- To access a protected property from a class that inhabit the class which the protected property is inside you need to use :  <span style="color:rgb(0, 176, 80)">NameofSuperClass</span>::NameofProperty
