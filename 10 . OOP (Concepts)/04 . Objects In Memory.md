# Objects in Memory

## Understanding Classes vs Objects

### Class: The Blueprint
A **class** is a template or blueprint that defines the structure and behavior of objects. It doesn't occupy memory by itself.

```
┌───────────────────────────┐
│   clsPerson (Blueprint)  │
│   ─────────────────────   │
│   • FirstName: string    │
│   • LastName: string     │
│   • GetFullName()        │
└───────────────────────────┘
```

### Object: The Instance
An **object** is a concrete instance of a class that actually exists in memory and stores data.

```
    Person1 (Object)          Person2 (Object)
┌────────────────────┐    ┌────────────────────┐
│ FirstName: "John"  │    │ FirstName: "Jane"  │
│ LastName: "Doe"    │    │ LastName: "Smith"  │
└────────────────────┘    └────────────────────┘
```

## Memory Allocation

When you create an object, memory is allocated **only for its data members**:

```
HEAP MEMORY:
┌─────────────────────────────────────────────┐
│                                             │
│  Person1:                                   │
│  ┌──────────────────────┐                   │
│  │ FirstName (string)   │ ← Memory space    │
│  │ LastName (string)    │ ← Memory space    │
│  └──────────────────────┘                   │
│                                             │
│  Person2:                                   │
│  ┌──────────────────────┐                   │
│  │ FirstName (string)   │ ← Memory space    │
│  │ LastName (string)    │ ← Memory space    │
│  └──────────────────────┘                   │
│                                             │
└─────────────────────────────────────────────┘

CODE SEGMENT (shared):
┌─────────────────────────────────────────────┐
│  GetFullName() method                       │
│  (shared by all clsPerson objects)          │
└─────────────────────────────────────────────┘
```

## Method Sharing

**Important**: Method members are **shared** among all objects of the same class. They exist once in memory, not duplicated for each object.

```
       Person1           Person2
         │                  │
         │                  │
         └─────────┬────────┘
                   │
                   ▼
         ┌──────────────────┐
         │  GetFullName()   │  ← Shared method
         └──────────────────┘
```

## Why Methods Are Shared

Each object has its own **data**, but the **code** (methods) is the same for all objects. This is memory efficient:

- ✅ **Data members**: Each object gets its own copy
- ✅ **Method members**: All objects share the same code

## Complete Example

```cpp
#include <iostream>
using namespace std;

class clsPerson
{
    public:
        // Data Members (each object has its own)
        string FirstName;
        string LastName;
    
        // Method Member (shared by all objects)
        string GetFullName()
        {
            return FirstName + " " + LastName;
        }
};

int main()
{
    // Creating Object 1
    clsPerson Person1;
    Person1.FirstName = "MOHAMMED";
    Person1.LastName = "Abu-Hadroud";
    
    // Creating Object 2
    clsPerson Person2;
    Person2.FirstName = "Walid";
    Person2.LastName = "Ahlidou";
    
    // Both objects use the same GetFullName() method
    // but operate on their own data
    cout << "Person 1: " << Person1.GetFullName() << endl;
    cout << "Person 2: " << Person2.GetFullName() << endl;
    
    return 0;
}
```

**Output:**
```
Person 1: MOHAMMED Abu-Hadroud
Person 2: Walid Ahlidou
```

## Key Concepts

| Aspect | Class | Object |
|--------|-------|--------|
| **Nature** | Blueprint/Template | Actual instance |
| **Memory** | No memory allocated | Memory allocated |
| **Data** | No data stored | Stores actual data |
| **Count** | One definition | Can create many |

## Memory Efficiency

```
❌ INEFFICIENT (if methods were duplicated):
Person1: [FirstName, LastName, GetFullName()]
Person2: [FirstName, LastName, GetFullName()]
Person3: [FirstName, LastName, GetFullName()]

✅ EFFICIENT (actual implementation):
Person1: [FirstName, LastName] ─┐
Person2: [FirstName, LastName] ─┼─→ [GetFullName()]
Person3: [FirstName, LastName] ─┘
```

## Important Notes

1. **Each object is independent** - changing Person1's data doesn't affect Person2
2. **Methods access the calling object's data** - when Person1 calls GetFullName(), it uses Person1's FirstName and LastName
3. **Memory is allocated on object creation** - using `clsPerson Person1;`
4. **The class itself doesn't occupy memory** - it's just a blueprint
