## Overview

A **Doubly Linked List** is a linear data structure that consists of a series of connected nodes, where each node has **two pointers** - one pointing to the next node and another pointing to the previous node. This bidirectional linking allows traversal in both forward and backward directions.

## Structure Comparison

### Singly Linked List
```
Head → [Data|Next] → [Data|Next] → [Data|Next] → NULL
```

### Doubly Linked List
```
     ┌─────────┐     ┌─────────┐     ┌─────────┐
NULL ← │Prev|Data│Next ← │Prev|Data│Next ← │Prev|Data│Next → NULL
     └─────────┘     └─────────┘     └─────────┘
       ↑ Head
```

## Node Structure

Each node in a doubly linked list contains three components:

1. **Data**: The actual value stored in the node
2. **Next Pointer**: Address/reference to the next node in the sequence
3. **Prev Pointer**: Address/reference to the previous node in the sequence

## Key Characteristics

### Advantages ✅

1. **Bidirectional Traversal**: Can move forward and backward through the list
2. **Easier Deletion**: Deleting a node is easier since we have access to the previous node
3. **Reverse Traversal**: Can traverse the list in reverse without recursion
4. **Insert Before**: Easy to insert a node before a given node

### Disadvantages ❌

1. **Extra Memory**: Requires additional memory for the previous pointer
2. **Complex Operations**: Insert and delete operations require more pointer updates
3. **More Maintenance**: Need to maintain two pointers instead of one

## Basic Operations

### 1. Insertion
- Insert at beginning
- Insert at end
- Insert at specific position

### 2. Deletion
- Delete from beginning
- Delete from end
- Delete specific node

### 3. Traversal
- Forward traversal (Head to Tail)
- Backward traversal (Tail to Head)

### 4. Search
- Find a node with specific data

## Comparison: Singly vs Doubly Linked List

| Feature | Singly Linked List | Doubly Linked List |
|---------|-------------------|-------------------|
| **Pointers per node** | 1 (Next) | 2 (Next & Prev) |
| **Traversal direction** | Forward only | Both directions |
| **Memory usage** | Less | More |
| **Deletion complexity** | Need previous node | Direct access to previous |
| **Implementation** | Simpler | More complex |
| **Reverse traversal** | Difficult | Easy |

## Common Use Cases

- **Browser History**: Back and forward navigation
- **Music Players**: Previous and next song functionality
- **Undo/Redo Operations**: In text editors and applications
- **LRU Cache**: Least Recently Used cache implementation
- **Navigation Systems**: Where bidirectional movement is needed

## Time Complexity

| Operation | Time Complexity |
|-----------|----------------|
| Access | O(n) |
| Search | O(n) |
| Insert at beginning | O(1) |
| Insert at end | O(1)* |
| Delete at beginning | O(1) |
| Delete at end | O(1)* |

*O(1) if we maintain a tail pointer

## Key Takeaway

A doubly linked list is ideal when you need **bidirectional traversal** and **frequent insertions/deletions** at both ends. The trade-off is extra memory for the additional pointer, but this cost is often worth it for the added flexibility and efficiency in certain operations.
