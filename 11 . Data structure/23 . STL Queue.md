# Understanding STL Queue in C++

## What is a Queue? ðŸŽ¯

A **queue** is a linear data structure that follows the **FIFO (First In, First Out)** principle. Think of it like a line of people waiting at a ticket counter:
- The first person to join the line is the first to be served
- New people join at the back of the line
- People leave from the front of the line

## How Queue Works - Step by Step

### Starting with an Empty Queue
```
Queue: [ ]
```

### After `push(10)`
```
Queue: [ 10 ]
        Front & Back
```

### After `push(20)`
```
Queue: [ 10 | 20 ]
        Front   Back
```

### After `push(30)` and `push(40)`
```
Queue: [ 10 | 20 | 30 | 40 ]
        Front              Back
```

### After `pop()` - Removes 10
```
Queue: [ 20 | 30 | 40 ]
        Front         Back
```

## Key STL Queue Functions

| Function | Description | Example Output |
|----------|-------------|----------------|
| `push(value)` | Adds element to the back | - |
| `pop()` | Removes element from the front | - |
| `front()` | Returns the front element | `10` |
| `back()` | Returns the back element | `40` |
| `size()` | Returns number of elements | `4` |
| `empty()` | Checks if queue is empty | `true/false` |

## Code Breakdown

### 1. Include the Queue Library
```cpp
#include <queue>
```

### 2. Create a Queue
```cpp
queue<int> MyQueue;  // Queue of integers
```

### 3. Add Elements (Enqueue)
```cpp
MyQueue.push(10);  // [10]
MyQueue.push(20);  // [10, 20]
MyQueue.push(30);  // [10, 20, 30]
MyQueue.push(40);  // [10, 20, 30, 40]
```

### 4. Access Elements
```cpp
MyQueue.front();  // Returns 10 (first element)
MyQueue.back();   // Returns 40 (last element)
MyQueue.size();   // Returns 4 (total elements)
```

### 5. Remove Elements (Dequeue)
```cpp
while (!MyQueue.empty()) {
    cout << MyQueue.front() << " ";  // Print front element
    MyQueue.pop();                    // Remove front element
}
// Output: 10 20 30 40
```

## Important Notes âš ï¸

1. **No Direct Access**: Unlike arrays, you cannot access elements in the middle of a queue
2. **pop () Returns Nothing**: `pop()` only removes the element, it doesn't return it. Use `front()` first to get the value
3. **Always Check Empty**: Before calling `front()`, `back()`, or `pop()`, ensure the queue is not empty

## Real-World Applications ðŸŒ

- **Print Queue**: Documents waiting to be printed
- **Task Scheduling**: CPU scheduling processes
- **Breadth-First Search**: Graph traversal algorithms
- **Customer Service**: Managing waiting customers
- **Message Queues**: Handling messages in order

## Practice Example

```cpp
queue<string> ticketLine;

// People joining the line
ticketLine.push("Alice");
ticketLine.push("Bob");
ticketLine.push("Charlie");

    cout << "Serving: " << ticketLine.front() << endl;
    ticketLine.pop();
}

// Output:
// Serving: Alice
// Serving: Bob
// Serving: Charlie
```

## Queue vs Stack

| Feature | Queue | Stack |
|---------|-------|-------|
| Principle | FIFO (First In, First Out) | LIFO (Last In, First Out) |
| Insertion | Back (push) | Top (push) |
| Removal | Front (pop) | Top (pop) |
| Access | front () & back () | top () |
| Analogy | Waiting line | Stack of plates |
