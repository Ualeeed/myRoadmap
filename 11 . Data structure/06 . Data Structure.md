### Data Structures vs Algorithms

**Data Structures**: Data Structures are used to organise and store data to use it in an effective way when performing data operations.

**Algorithms**: Algorithm is a step-by-step procedure, which defines a set of instructions to be executed in a certain order to get the desired output.

**Simple explanation**: 
- Data structures are like containers (boxes, shelves, filing cabinets)
- Algorithms are like instructions (recipes, directions, procedures)
- You need the right container AND the right instructions to solve problems efficiently

## ðŸ“Š Types of Data Structures

Data structures are divided into two main categories:

### 1. Linear Data Structures
Elements are arranged in a sequence, one after another.

#### **Arrays**
Array is a linear data structure where elements are allocated contiguous memory, allowing for constant-time access.

**Think of it as**: A row of numbered mailboxes
- Each box (element) has a number (index)
- All boxes are the same size
- You can instantly access any box if you know its number

**Example Uses**:
- Storing a list of student grades
- Managing a playlist of songs
- Storing pixels in an image

#### **Linked Lists**
A linear data structure that stores data in nodes, which are connected by pointers. Unlike arrays, nodes of linked lists are not stored in contiguous memory locations and can only be accessed sequentially, starting from the head of list.

**Think of it as**: A treasure hunt with clues
- Each node holds data AND directions to the next node
- More flexible than arrays for inserting/removing items
- Must follow the chain from start to reach any item

**Example Uses**:
- Browser history (back/forward buttons)
- Music playlist where you can easily add/remove songs
- Undo functionality in applications

#### **Stacks**
A linear data structure that follows the Last In, First Out (LIFO) principle. Stacks play an important role in managing function calls, memory, and are widely used in algorithms.

**Think of it as**: A stack of plates
- Add plates on top (push)
- Remove plates from top (pop)
- Last plate you put is the first one you take

**Example Uses**:
- Undo/Redo operations
- Browser back button
- Expression evaluation in calculators
- Function call management

#### **Queues**
Queue is a linear data structure that follows the First In, First Out (FIFO) principle. Queues play an important role in managing tasks or data in order, scheduling and message handling systems.

**Think of it as**: A line at a coffee shop
- First person in line gets served first
- New people join at the back
- Fair ordering system

**Example Uses**:
- Print job queue
- Customer service ticket systems
- Playlist management
- Message handling systems

### 2. Non-Linear Data Structures
Elements are arranged hierarchically or with complex relationships.

#### **Trees**
A non-linear, hierarchical data structure consisting of nodes connected by edges, with a top node called the root and nodes having child nodes.

**Think of it as**: A family tree or organization chart
- One root (top person/node)
- Parent-child relationships
- Branches spreading out

**Example Uses**:
- File system organization (folders and files)
- HTML DOM structure
- Decision-making systems
- Database indexing

#### **Graphs**
A non-linear data structure consisting of a finite set of vertices(or nodes) and a set of edges(or links)that connect a pair of nodes. Graphs are widely used to represent relationships between entities.

**Think of it as**: A social network or road map
- Many nodes (people/cities) connected in various ways
- Relationships can go in any direction
- No single starting point

**Example Uses**:
- Social networks (Facebook friends)
- Maps and navigation systems
- Network topology
- Recommendation systems

## ðŸ” Important Concepts

### Hashing
Hashing is a technique that generates a fixed-size output (hash value) from an input of variable size using mathematical formulas called hash functions. Hashing is commonly used in data structures for efficient searching, insertion and deletion.

**Think of it as**: A library card catalog
- Convert book title to a specific location
- Find books super fast
- No need to search through every book

### Searching Algorithms
Searching algorithms are used to locate specific data within a large set of data. It helps find a target value within the data.

Common types:
- **Linear Search**: Check each item one by one
- **Binary Search**: Divide and conquer (only works on sorted data)

### Sorting Algorithms
Sorting algorithms are used to arrange the elements of a list in a specific order, such as numerical or alphabetical. It organizes the items in a systematic way, making it easier to search for and access specific elements.

Common types:
- **Bubble Sort**: Compare adjacent items and swap
- **Quick Sort**: Divide into smaller parts
- **Merge Sort**: Divide, sort, then merge
