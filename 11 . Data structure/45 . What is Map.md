# What is Map

## Overview
A **Map** is a data structure that stores data in **key-value pairs**, where each key is unique and maps to exactly one value. Think of it like a real dictionary where you look up a word (key) to find its definition (value).

## Simple Analogy
Imagine a phonebook:
- **Key**: Person's name (must be unique)
- **Value**: Their phone number

You search by name (key) and get the corresponding phone number (value).

## Key Characteristics

### 1. **Key-Value Pairs**
- Data is stored as pairs: `(key, value)`
- Example: `("username", "john123")` or `(1, "Apple")`

### 2. **Unique Keys**
- Each key can appear only once in the map
- Keys act as unique identifiers
- Values can be duplicated, but not keys

### 3. **Sorted Order (in C++ std::map)**
- Keys are automatically sorted in ascending order
- This enables fast searching and retrieval
- Uses a balanced tree structure internally (typically Red-Black Tree)

### 4. **Fast Operations**
- **Search**: Quickly find a value using its key
- **Insert**: Add new key-value pairs efficiently
- **Delete**: Remove entries by key
- Time Complexity: O(log n) for most operations

## Map vs Dictionary

In many programming languages, **Map** and **Dictionary** refer to the same concept:
- **C++**: Called `std::map`
- **Python**: Called `dict` (dictionary)
- **JavaScript**: Called `Object` or `Map`
- **Java**: Has both `HashMap` and `TreeMap`

All serve the same purpose: storing key-value pairs for fast lookup.

## Basic Operations

### 1. **Insert/Add**
Add a new key-value pair to the map
```
map["key"] = value
```

### 2. **Access/Retrieve**
Get the value associated with a key
```
value = map["key"]
```

### 3. **Update**
Change the value of an existing key
```
map["existingKey"] = newValue
```

### 4. **Delete/Remove**
Remove a key-value pair from the map
```
map.remove("key")
```

### 5. **Search/Contains**
Check if a key exists in the map
```
if map.contains("key"):
    # Key exists
```

## How It Works Internally (C++ std::map)

1. **Tree Structure**: Uses a self-balancing binary search tree (Red-Black Tree)
2. **Automatic Sorting**: Keys are kept in sorted order
3. **Logarithmic Time**: Most operations take O(log n) time
4. **Memory Nodes**: Each key-value pair is stored in a tree node

```
Internal Tree Structure:
        [Key: "M"]
       /          \
  [Key: "D"]    [Key: "T"]
    /    \        /    \
  [...]  [...]  [...] [...]
```

## Map Variations

### 1. **Ordered Map** (C++ `std::map`)
- Keys are sorted
- Uses tree structure
- O(log n) operations

### 2. **Unordered Map** (C++ `std::unordered_map`)
- Keys are not sorted
- Uses hash table
- O(1) average operations
- Faster but no order guarantee

### 3. **Multi-Map**
- Allows duplicate keys
- One key can map to multiple values

## Complexity Comparison

| Operation | std::map (Tree) | std::unordered_map (Hash) |
|-----------|----------------|---------------------------|
| Search    | O(log n)       | O(1) average              |
| Insert    | O(log n)       | O(1) average              |
| Delete    | O(log n)       | O(1) average              |
| Space     | O(n)           | O(n)                      |

## Summary

A **Map** is like a smart lookup table that stores data as key-value pairs. It's similar to a dictionary where you look up words (keys) to find their meanings (values). The map automatically keeps keys sorted and provides fast search, insertion, and deletion operations. It's one of the most useful data structures for organizing and retrieving data efficiently.

**Key Takeaway**: When you need to quickly find data using a unique identifier, a map is your go-to data structure!
