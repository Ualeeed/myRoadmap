- Write a program to <span style="color:rgb(255, 31, 31)">check if the matrices are typical or not</span> : 

```cpp
#include <iostream>
#include <string>
#include <iomanip>

using namespace std;


int RandomNumber(int From, int To)
{
	//Function to generate a random number
	int randNum = rand() % (To - From + 1) + From;
	return randNum;
}


void FillMatrixWithRandomNumbers(int arr[3][3], short Rows, short Cols)
{

	for (short i = 0; i < Rows; i++)
	{
		for (short j = 0; j < Cols; j++)
		{
			arr[i][j] = RandomNumber(1, 10);
		}
	}
}


void PrintMatrix(int arr[3][3], short Rows, short Cols)
{
	for (short i = 0; i < Rows; i++)
	{
		for (short j = 0; j < Cols; j++)
		{
			printf(" %0*d ", 2, arr[i][j]);
			//cout << setw(3) << arr[i][j] << " ";
		}
		cout << "\n";
	}
}


bool AreTypicalMatrices(int Matrix1[3][3], int Matrix2[3][3],
	short Rows, short Cols)
{
	for (short i = 0; i < Rows; i++)
	{
		for (short j = 0; j < Cols; j++)
		{
			if (Matrix1[i][j] != Matrix2[i][j])
			{
				return false;
			}
		}
	}
	return true;
}


int main()
{

	//Seeds the random number generator in C++, called only once
	srand((unsigned)time(NULL));


	int Matrix1[3][3], Matrix2[3][3];


	FillMatrixWithRandomNumbers(Matrix1, 3, 3);


	cout << "\nMatrix1:\n";

	PrintMatrix(Matrix1, 3, 3);


	FillMatrixWithRandomNumbers(Matrix2, 3, 3);


	cout << "\nMatrix2:\n";

	PrintMatrix(Matrix2, 3, 3);


	if (AreTypicalMatrices(Matrix1, Matrix2, 3, 3))
		cout << "\nYES: both are typical.";
	else
		cout << "\nNo: they are NOT typical.";


	system("pause>0");
}

```

## Explanation

This program implements **true matrix equality** by comparing each corresponding element between two matrices. The term "typical matrices" appears to mean "identical matrices".

### Key Concepts:

**True Matrix Equality:**
- Two matrices are equal if and only if they have the same dimensions AND each corresponding element is identical
- Mathematical definition: A = B if A[i][j] = B[i][j] for all valid i,j
- This is the standard and correct way to compare matrices

### Algorithm Breakdown:

1. **Matrix Generation:** Creates two separate 3x3 matrices with random numbers

2. **Element-by-Element Comparison (`AreTypicalMatrices`):**
   - Uses nested loops to traverse both matrices simultaneously
   - Compares each `Matrix1[i][j]` with `Matrix2[i][j]`
   - **Early Exit Optimization:** Returns `false` immediately when first difference is found
   - Only returns `true` if ALL elements match exactly

3. **Efficient Logic:** Stops checking as soon as a mismatch is detected

### Example Scenarios:

**Scenario 1 - Identical Matrices:**
```
Matrix1:        Matrix2:
 5  3  8         5  3  8
 2  7  1         2  7  1
 9  4  6         9  4  6

Result: "YES: both are typical" (identical)
```

**Scenario 2 - Different Matrices:**
```
Matrix1:        Matrix2:
 5  3  8         5  3  2  ← Different value here
 2  7  1         2  7  1
 9  4  6         9  4  6

Result: "No: they are NOT typical" (different)
```

### Why This is Better Than Sum Comparison:
- **Precise:** No false positives like sum-based comparison
- **Standard:** Follows mathematical definition of matrix equality
- **Reliable:** Detects any structural differences

### Performance Optimization:
- **Best Case:** O(1) - difference found at first element
- **Worst Case:** O(m×n) - matrices are identical or differ at last element
- **Average Case:** O(m×n/2) - difference found roughly halfway through

### Real-world Applications:
- Data validation and verification
- Image comparison (pixels as matrix elements)
- Scientific computation result verification
- Database record comparison

### Time Complexity: O(m×n) worst case, O(1) best case
### Space Complexity: O(1) - only using loop variables
